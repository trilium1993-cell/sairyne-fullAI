Universal Layout Normalizer for Anima → Cursor Integration

Goal:
Whenever Anima-generated code is imported into Cursor (from Figma), 
this prompt standardizes all layout, spacing, and styles to ensure the rendered result in Cursor 
matches the original Anima preview pixel-for-pixel while remaining responsive and React-compliant.

---

Tasks:

1️⃣ CLEAN STRUCTURE
- Remove ALL inline styles generated by Anima.
- Remove all `position: absolute`, `top`, `left`, `right`, `bottom` declarations unless required for icons or overlays.
- Replace absolute positioning with responsive layout using Tailwind flex or grid.
- Remove unnecessary wrapper divs created by Anima (keep semantic grouping only).

---

2️⃣ FLEXIBLE LAYOUT
- Convert every parent container with multiple children to:
  ```jsx
  className="flex flex-col items-start justify-start gap-4 w-full"
	•	If horizontal alignment is intended → flex-row items-center gap-4
	•	Respect original Figma spacing by using Tailwind gap, padding, and margin utilities instead of pixel-based absolute values.
	•	Centered blocks → mx-auto
	•	Avoid fixed pixel widths unless essential; prefer max-w-[value] with relative widths.

⸻

3️⃣ TYPOGRAPHY & COLORS
	•	Replace Anima inline font definitions with global, consistent typography:
ClassName="text-white text-[15px] leading-relaxed"
	•	Titles (h1/h2/h3):
className="text-[18px] font-semibold text-white mb-3"
	•	Subtext / captions:
className="text-white/80 text-[14px] mb-2"

	•	Do not use inline color codes from Anima; use global theme or Tailwind color tokens instead.

⸻

4️⃣ IMAGES
	•	Normalize all <img> tags:
<img
  src={imageSrc}
  alt=""
  className="w-full max-w-[560px] h-auto object-contain rounded-md mb-4"
/>

	•	Ensure all images align left (no center alignment unless explicit).
	•	Maintain consistent spacing between text and images (12–16px).

⸻

5️⃣ CONTAINER ALIGNMENT
	•	Make sure imported components align perfectly with the left chat baseline if used inside a two-column layout.
	•	Outer container:
className="flex flex-row w-full h-full items-start justify-between"

	•	Left panel: flex-1 h-full
	•	Right panel (e.g., Visual Tips):
className="flex flex-col w-[45%] max-w-[600px] h-full overflow-y-auto px-6 py-4 gap-4"

⸻

6️⃣ REMOVE ABSOLUTE SCALES FROM ANIMA EXPORT
	•	Delete or neutralize style attributes like:
	•	style="width:XXXpx; height:YYYpx;"
	•	transform, translate, rotate, or any CSS transform-based placement.
	•	Use Tailwind spacing (e.g., px-6 py-4, gap-4, mt-3, mb-3) instead.

⸻

7️⃣ GLOBAL BASELINE (include once in globals.css)
Add or confirm:
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

html, body {
  height: 100%;
  background: #0b0b0f;
  font-family: 'Inter', sans-serif;
  color: #ffffff;
  scroll-behavior: smooth;
}
Ensure that Google Fonts or local fonts from Figma are properly linked.

⸻

8️⃣ CONSISTENCY CHECK
	•	All sections must use gap-4 as the default vertical rhythm.
	•	Line height = 1.5em.
	•	No element should visually touch screen edges.
	•	Maintain horizontal balance between left and right columns.
	•	The layout must remain stable (no shifts) when re-rendering inside Cursor.

⸻

9️⃣ OPTIONAL SMOOTH APPEARANCE (for polish)
If the component is toggleable (like “Visual Tips”):
<motion.div
  initial={{ opacity: 0, x: 40 }}
  animate={{ opacity: 1, x: 0 }}
  exit={{ opacity: 0, x: 40 }}
  transition={{ duration: 0.25 }}
>
Wrap it in <AnimatePresence> for smooth entry/exit.

⸻

✅ Acceptance Criteria
	•	The rendered component in Cursor looks identical to the Anima preview.
	•	Text and images remain proportionate and aligned.
	•	No overlapping, no misaligned gaps, no extra wrappers.
	•	The component remains responsive and pixel-stable on all screens.
Whenever you import new Anima code:
→ Paste it into Cursor.
→ Run this prompt.
→ Cursor should refactor it automatically to follow these rules.
